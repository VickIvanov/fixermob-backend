# Решение проблем с доступом к серверу

## Сервер запущен и работает

Если вы не можете открыть `http://localhost:5001` в браузере, попробуйте следующее:

### 1. Проверьте, что сервер запущен

В терминале выполните:
```bash
lsof -i :5001
```

Должен быть виден процесс Python, слушающий порт 5001.

### 2. Попробуйте разные варианты URL

- `http://localhost:5001`
- `http://127.0.0.1:5001`
- `http://0.0.0.0:5001`

### 3. Проверьте через curl

```bash
curl http://localhost:5001/api/health
```

Если curl работает, но браузер не открывает - проблема может быть в браузере или настройках системы.

### 4. Проверьте брандмауэр

На macOS проверьте настройки брандмауэра в:
**Системные настройки → Безопасность → Брандмауэр**

### 5. Перезапустите сервер

Остановите текущий процесс:
```bash
pkill -f "python app.py"
```

Запустите заново:
```bash
cd /Users/vick/PycharmProjects/fixermob-backend
source venv/bin/activate
python app.py
```

### 6. Проверьте логи сервера

При запуске сервера в терминале вы должны видеть:
```
 * Serving Flask app 'app'
 * Debug mode: on
 * Running on http://0.0.0.0:5001
```

### 7. Альтернативный способ запуска

Если проблемы продолжаются, попробуйте запустить сервер на другом порту:

В файле `app.py` измените:
```python
app.run(host='127.0.0.1', port=8000, debug=True)
```

Затем откройте `http://localhost:8000`

### 8. Проверка через Python

Создайте тестовый скрипт `test_server.py`:
```python
import requests
try:
    response = requests.get('http://localhost:5001/api/health')
    print(f"Статус: {response.status_code}")
    print(f"Ответ: {response.json()}")
except Exception as e:
    print(f"Ошибка: {e}")
```

Запустите:
```bash
python test_server.py
```

## Если ничего не помогает

1. Убедитесь, что порт 5001 не занят другим приложением
2. Попробуйте перезагрузить компьютер
3. Проверьте, нет ли проблем с сетевой конфигурацией macOS

## Контакты для отладки

Если проблема сохраняется, проверьте:
- Версию Python: `python --version` (должна быть 3.8+)
- Установленные пакеты: `pip list`
- Логи сервера в терминале, где он запущен
